# Nightshift: Logging Quality Auditor

## Executive Summary

Analyzed 3 Python files (731 lines total) in hermes-nightshift-glm. This utility project (nightshift orchestrator + GLM quota checker) uses `print()` for all output with no structured logging. No error context is captured, and failures are silent.

## Findings

### P1 — High

- **glm_quota.py** — No `logging` module imported. All output via `print()`. No log levels, no structured fields, no way to filter verbosity.
- **nightshift.py** — No `logging` module imported. All output via `print()`. Status messages and error conditions indistinguishable.
- **Both files** — `sys.exit()` calls on error without logging the reason programmatically. Errors are printed to stdout (not stderr), making them invisible to log collectors.

### P2 — Medium

- **nightshift.py** — HTTP errors from `gh` CLI and git commands caught generically with no structured error info. Just prints and continues.
- **glm_quota.py** — `_fetch()` catches urllib errors but only returns None on failure — callers can't distinguish "API down" from "quota fully consumed".

### P3 — Low

- **Both files** — No log rotation or log file configuration. All output goes to stdout.
- **nightshift.py** — `random.shuffle()` used for task selection with no seed logging, making run reproduction impossible.

## Recommendations

1. **Import `logging` module** in both files, configure with `logging.basicConfig()`
2. **Replace `print()` with `logger.info()`/`logger.warning()`/`logger.error()`**
3. **Use stderr for errors** — `print(..., file=sys.stderr)` or `logger.error()`
4. **Add structured context** — log function names, parameters, HTTP status codes
5. **Log before `sys.exit()`** — capture exit reason at ERROR level
6. **Add verbose/quiet flags** — control log level via CLI args
